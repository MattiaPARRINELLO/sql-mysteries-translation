Mystère du meurtre SQL

Peux-tu découvrir le coupable

Il y a eu un meutre dans la ville SQL! Le mystère du meurtre SQL est designé pour être à la fois une leçon en autonomie pour apprendre les conceptes et commandes du SQL et pour être un jeu amusant pour les utilisateurs plus avancés de SQL et ainsi résoudre un crime.

Guide pour les nouveaux au SQL

Cet exercise est pensé pour pratiquer les compétences en SQL plutôt qu'un tutoriel complet. Si tu n'as jamais utilisé SQL, essayez le tutoriel. Si tu veux vraiment apprendre beaucoup sur SQL, tu préféres peut-être un tutoriel complet comme Select Star SQL.

Si tu es à l'aise avec SQL, tu peux plonger ci-dessous!

Expérimenté en SQL, commences ici

Un crime a eu lieu et le détective a besoin de votre aide. Le détective t'as donné le rapport de la scène de crime, mais tu l'as perdu. Tu te rappel vaguement que le crime était un meurtre qui a eu lieu à un moment donné le 15 janvier 2018 et qu'il a eu lieu à SQL City. Commence par récupérer le rapport de la scène de crime correspondant dans la base de données du département de police.

Exploration de la structure de la base de données

Les utilisateurs avancés de SQL peuvent souvent utiliser des requêtes de base de données pour déduire la structure d'une base de données. Mais chaque système de base de données a des façons différentes de gérer ces informations. Le Mystère du meurtre SQL est construit en utilisant SQLite. Utilisez cette commande SQL pour trouver les tables dans la base de données du Mystère du meurtre.

En plus de connaître les noms des tables, vous devez connaître la structure de chaque table. La façon dont cela fonctionne dépend également de la technologie de base de données que vous utilisez. Voici comment vous faites avec SQLite.
